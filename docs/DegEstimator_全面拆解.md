## 一、引言：为什么需要 DegEstimator？

在视频压缩伪影去除（MFQEv2）任务中，A方案（PureResUNet）存在一个根本性的盲区：**它对输入帧的压缩强度"浑然不觉"**。

无论输入是 QP22（轻微压缩）还是 QP42（重度压缩），A方案都使用同一套卷积权重、同一套修复策略。这就像一位医生，不管病人是感冒还是骨折，都开同一种药。

**DegEstimator 的出现，就是为了给网络加一只"眼睛"**——让它在修复之前，先"看"一眼输入帧的退化程度，从而决定"修多狠"。

---

## 二、设计哲学：三个关键选择

DegEstimator 的设计不是拍脑袋决定的，它背后有三个经过反复权衡的选择。

### 2.1 选择一：隐式编码，而非显式预测

**显式预测的做法**是：让网络输出一个具体的 QP 数值（比如预测"这是一帧 QP37 的图像"），然后把 QP 数值作为条件输入后续模块。  
**隐式编码的做法**是：让网络输出一个高维向量，这个向量"编码"了退化信息，但人类无法直接解读它具体对应哪个 QP。

我们选择了**隐式编码**，理由来自 PromptCIR (CVPRW 2024) 的核心洞察：

> "预测数值 QF 缺乏空间信息，阻碍网络对图像内容的适应性。"

换句话说，**QP 只是一个粗糙的全局标量**。同一帧图像里，平坦区域和复杂纹理区域的退化感知应该不同；同一个 QP 下，不同序列的压缩特征也有差异。一个64维的嵌入向量，比单个 QP 数字能承载更丰富的、内容感知的退化信息。

### 2.2 选择二：端到端联合训练，而非预训练+冻结

**预训练+冻结的做法**是：先用一个独立任务（比如 QP 分类）训练 DegEstimator，然后把它的权重冻住，只训练主干网络。  
**联合训练的做法**是：DegEstimator 和主干网络一起初始化、一起前向、一起反向传播、一起更新。

我们选择了**联合训练**，理由是：

1. **梯度必须打通**：如果 DegEstimator 被冻住，它输出的退化嵌入就是"死"的。网络无法根据主任务（伪影去除）的反馈来调整"退化"的定义。比如，某些对修复有益的压缩特征可能在预训练任务里根本不是重点。
2. **任务耦合**：DegEstimator 不需要"通用"的退化表征，它只需要对**伪影去除任务有用**的退化表征。联合训练让 DegEstimator 自动学习"什么信息对修复有帮助"，而不是"什么信息能准确预测 QP"。
3. ** simplicity**：少了一个预训练阶段，少了一个超参数（冻结比例），pipeline 更干净。

### 2.3 选择三：轻量到可以忽略，而非越大越好

DegEstimator 的总参数量只有 **~0.1M**（代码实现版本），占 B 方案总参数（~12.36M）的 **0.8%**。

我们刻意保持轻量，因为：

1. **旁路定位**：DegEstimator 不是主干网络，它只是提供"辅助信息"。如果旁路太重，喧宾夺主，训练和推理开销都会增加。
2. **信息瓶颈**：退化信息本质上是**全局低维**的（一帧图像只有一个压缩强度）。用一个庞大的网络去提取一个标量/向量，是严重的 over-parameterization。
3. **速度敏感**：视频增强的推理速度至关重要。DegEstimator 只运行一次，但其计算量必须小到对整体 latency 无感。

## 三、架构逐层拆解

```
Input (B, 3, H, W)
    │
    ▼ Conv2d(3→32, 3×3, stride=2, padding=1, bias=False)
    │   + BatchNorm2d(32) + ReLU
    │
    Feature (B, 32, H/2, W/2)
    │
    ▼ Conv2d(32→64, 3×3, stride=2, padding=1, bias=False)
    │   + BatchNorm2d(64) + ReLU
    │
    Feature (B, 64, H/4, W/4)
    │
    ▼ Conv2d(64→128, 3×3, stride=2, padding=1, bias=False)
    │   + BatchNorm2d(128) + ReLU
    │
    Feature (B, 128, H/8, W/8)
    │
    ▼ AdaptiveAvgPool2d(1)
    │
    Feature (B, 128, 1, 1)
    │
    ▼ Flatten → Linear(128 → 64)
    │
    DegEmbed d (B, 64)
```

### 3.1 第一层下采样：3 → 32，stride=2

```python
nn.Conv2d(3, 32, 3, stride=2, padding=1, bias=False)
nn.BatchNorm2d(32)
nn.ReLU(inplace=True)
```

- **输入**：原始 RGB 帧 (B, 3, H, W)
- **输出**：(B, 32, H/2, W/2)
- **参数**：3 × 32 × 3 × 3 = **864**
- **作用**：从原始像素提取初级特征，同时把分辨率降一半，减少后续计算量。
- **为什么 stride=2**：下采样不仅是为了提速，更是为了**扩大感受野**。退化信息（块效应、振铃）往往是大尺度的周期性模式，需要足够的感受野才能被感知。
- **为什么 bias=False**：因为后面紧跟 BatchNorm，BN 的 β 已经起到了 bias 的作用，去掉 bias 可以减少参数量并稳定训练。

### 3.2 第二层下采样：32 → 64，stride=2

```python
nn.Conv2d(32, 64, 3, stride=2, padding=1, bias=False)
nn.BatchNorm2d(64)
nn.ReLU(inplace=True)
```

- **输出**：(B, 64, H/4, W/4)
- **参数**：32 × 64 × 3 × 3 = **18,432**
- **作用**：通道数翻倍，空间分辨率再减半。网络开始提取更抽象的退化特征，比如块边界的位置信息、压缩伪影的频谱特征。

### 3.3 第三层下采样：64 → 128，stride=2

```python
nn.Conv2d(64, 128, 3, stride=2, padding=1, bias=False)
nn.BatchNorm2d(128)
nn.ReLU(inplace=True)
```

- **输出**：(B, 128, H/8, W/8)
- **参数**：64 × 128 × 3 × 3 = **73,728**
- **作用**：最深的一层。此时特征图的空间尺寸已经只有输入的 1/8，但通道数达到 128。这一层负责把局部的退化模式聚合成全局的语义级退化表征。
- **导师反馈的影响**：最初考虑过用 4 层下采样（到 1/16），但导师指出 1/16 会丢失块效应的周期结构。因此我们停在 1/8。

### 3.4 全局平均池化：AdaptiveAvgPool2d(1)

```python
nn.AdaptiveAvgPool2d(1)
```

- **输入**：(B, 128, H/8, W/8)
- **输出**：(B, 128, 1, 1)
- **参数**：**0**
- **作用**：把空间维度"压扁"成 1×1。这一步是 DegEstimator 从"空间特征提取器"转变为"全局退化编码器"的关键。
- **为什么用 Average Pooling 而不是 Max Pooling**：退化信息是**全局分布**的（整帧图像的压缩强度大致均匀），Average Pooling 能更好地聚合全局统计量。Max Pooling 会偏向于最显眼的局部伪影，可能丢失全局压缩强度的信息。
- **为什么不直接用 GAP 代替第三层 Conv**：因为原始分辨率太高（如 1920×1080），直接对原始图像做 GAP 会让浅层网络几乎没有提取特征的机会。先下采样到 1/8，再做 GAP，是特征提取和空间压缩的平衡。

### 3.5 全连接层：Linear(128 → 64)

```python
nn.Linear(128, 64)
```

- **输入**：(B, 128)（经过 Flatten）
- **输出**：(B, 64)
- **参数**：128 × 64 + 64 = **8,256**
- **作用**：把 128 维的 CNN 特征投影到 64 维的退化嵌入空间。这个投影是**可学习的降维**，让 DegEstimator 可以自动决定哪 64 个维度对后续修复任务最有用。
- **为什么输出 64 维**：这是一个经验值。太小（如 8 维）可能信息不够；太大（如 256 维）会增加 FiLM 生成器的负担（每个 FiLM 层需要一个 Linear(256 → 2C)）。64 维在表达能力和计算开销之间取得了平衡。

## 五、训练动态：它是怎么被"教"出来的？

DegEstimator 不是孤立训练的，它的学习过程深深嵌入在 B 方案的整体训练中。

### 5.1 前向传播流程

```python
deg_embed = self.deg_estimator(identity)  # (B, 64)

# Bottleneck 第一层
film_params = self.film_bottleneck[0](deg_embed)  # Linear(64 → 512)
x = self.bottleneck[0](x, film_params)

# Bottleneck 第二层
film_params = self.film_bottleneck[1](deg_embed)
x = self.bottleneck[1](x, film_params)

# Decoder 各层...
```

注意：**DegEstimator 的输入是 `identity`（原始输入帧），而不是主干网络提取的特征**。这是一个重要设计：

- 如果 DegEstimator 接收主干特征，它会看到已经被 init_conv 和 Encoder 处理过的特征，这些特征已经被"内容化"了，退化信息可能被稀释。
- 让 DegEstimator 直接看原始像素，确保它接收的是**最原始、最完整**的退化信号。

### 5.2 反向传播与梯度流

当损失函数 `L = L1_SSIM(pred, gt)` 反向传播时，梯度会沿着两条路径回流：

```
路径1: pred → Decoder → Bottleneck → Encoder → InitConv (主干网络)
路径2: pred → Decoder → FiLM → film_bottleneck → deg_embed → DegEstimator
```

**DegEstimator 的梯度来自 FiLM 层**：

```
∂L/∂deg_embed = Σ (∂L/∂film_params_i) × (∂film_params_i/∂deg_embed)
```

其中 `i` 遍历所有 FiLM 层（Bottleneck 2层 + Decoder 6层 = 8层）。这意味着 DegEstimator 接收的是**所有 FiLM 层的梯度聚合**，它会被告知："你应该输出什么样的退化嵌入，才能让这些 FiLM 层更好地帮助修复。"

### 5.3 训练初期的行为

在训练刚开始时（epoch 0~5），DegEstimator 是"随机"的，FiLM 层的输出 γ≈0, β≈0（因为权重初始化为0）。此时整个 B 方案的行为几乎等同于 A 方案。

随着训练进行：
- 主干网络先学会基本的修复能力（和 A 方案一样）。
- FiLM 层开始微调残差出力。
- DegEstimator 逐渐学会区分不同 QP 的输入，输出不同的退化嵌入。
- 三者形成**协同进化**：DegEstimator 输出更好的嵌入 → FiLM 调节更精准 → 修复效果更好 → 梯度信号更清晰 → DegEstimator 学得更好。

## 六、与显式 QP 预测器的对比

为了说明隐式编码的优势，我们设计一个思想实验：假设不用 DegEstimator，而是直接输入 QP 数值。

| 维度 | 显式 QP 输入 | DegEstimator 隐式编码 |
|------|------------|---------------------|
| 输入形式 | 单个整数（如 32） | 64维连续向量 |
| 空间感知 | ❌ 全局一致，无空间差异 | ✅ 虽然最终是全局向量，但提取过程经历了空间卷积，隐含空间信息 |
| 内容感知 | ❌ QP22 永远是同一个数字 | ✅ 同一 QP 下，平坦区域和纹理区域产生的嵌入不同 |
| 跨数据集 | ❌ 不同编码器的 QP 定义不同 | ✅ 可以自适应学习不同编码器的退化特征 |
| 端到端优化 | ⚠️ QP 是离散的，不可微 | ✅ 完全可微，梯度畅通 |
| 额外监督 | 需要 QP 标签 | 不需要任何额外标签 |

## 七、消融与变体：如果改变 DegEstimator 会怎样？

以下是基于设计空间可以做的消融实验（部分尚未运行）：

### 7.1 消融1：去掉 DegEstimator，固定退化嵌入为0

预期结果：FiLM 层输出恒为 γ=0, β=0，B 方案退化为 A 方案。  
**验证价值**：证明 FiLM 层本身不会引入额外 capacity，性能差异确实来自退化感知。

### 7.2 消融2：冻结 DegEstimator，只训练主干

预期结果：性能介于 A 和 B 之间，但不如联合训练的 B。  
**验证价值**：证明联合训练的必要性——预训练的退化表征不如任务驱动的退化表征。

### 7.3 消融3：把 DegEstimator 换成显式 QP Embedding

做法：用一个 Embedding 层 `nn.Embedding(5, 64)`，直接把 QP22/27/32/37/42 映射到 64 维向量。  
预期结果：训练 QP（32）上可能不差，但跨 QP 泛化能力显著下降。  
**验证价值**：证明"隐式编码 > 显式标签"。

### 7.4 消融4：Encoder 也插入 FiLM

做法：让 DegEstimator 的嵌入也注入 Encoder。  
预期结果：可能轻微提升，也可能让浅层特征被退化信息"污染"，细节丢失。  
**验证价值**：验证"Encoder 不插"设计选择的合理性。

### 7.5 消融5：不同下采样深度

- 2层下采样（1/4）：感受野可能不够，难以捕获大尺度块效应。
- 4层下采样（1/16）：如导师所言，可能丢失块效应的周期结构。
- 3层下采样（1/8）：当前选择，平衡感受野和细节保留。

## 八、可视化：DegEstimator 到底"看"到了什么？

虽然 DegEstimator 输出的是 64 维向量，人类无法直接解读，但我们可以通过间接手段窥探它的行为。

### 8.1 方法1：t-SNE 降维可视化

取 test 集中不同 QP 的样本，提取它们的 `deg_embed`，用 t-SNE 降到 2D。

**预期现象**：
- 相同 QP 的样本在 2D 平面上聚成一堆。
- 不同 QP 的样本形成有序排列（如 QP22 → QP42 沿某个方向渐变）。
- 如果聚类不清晰，说明 DegEstimator 没有有效区分退化程度。

### 8.2 方法2：γ 分布统计

统计不同 QP 下，各 FiLM 层生成的 γ 的均值和方差。

**预期现象**：
- QP42 的 γ 均值 > QP22 的 γ 均值（高压缩时修复更积极）。
- 如果相反或不单调，说明 DegEstimator/FiLM 的协同机制出了问题。

### 8.3 方法3：敏感性分析

对同一帧图像，人为加入不同强度的压缩噪声，观察 `deg_embed` 的变化幅度。

**预期现象**：`deg_embed` 的 L2 范数应与压缩强度正相关。

## 九、局限性：这只"眼睛"的不完美之处

### 9.1 局限1：全局性假设

DegEstimator 最终输出的是一个**全局向量**（batch-wise, 每帧一个64维向量）。这意味着它假设整帧图像的退化强度是均匀的。但在实际视频中，可能存在局部质量变化（比如 I 帧和 P 帧的差异、码率波动导致的局部模糊）。

**改进方向**：让 DegEstimator 输出空间特征图（如保持 H/8 × W/8 的分辨率），实现空间自适应的退化感知。

### 9.2 局限2：无时间感知

DegEstimator 只看当前帧，不看前后帧。但视频压缩的伪影往往有时域相关性（比如运动补偿残差导致的闪烁）。如果时间信息能被纳入退化估计，可能会更精准。

**改进方向**：引入时域卷积或光流对齐，让 DegEstimator 看到相邻帧。

### 9.3 局限3：通道级 FiLM 的调节粒度有限

当前 FiLM 只做通道级的缩放和平移（γ·F + β），没有空间选择性。这意味着 DegEstimator 只能说"这一帧要多修一点"，但不能说"这一帧的左上角要多修，右下角少修"。

**改进方向**：升级为空间-通道联合调制（如 STDA、Spatial-FiLM），或者引入注意力机制。

### 9.4 局限4：与主干共享输入的潜在冲突

DegEstimator 和主干网络都接收原始输入帧，但学习目标不同：
- 主干想提取"内容特征"（边缘、纹理、语义）。
- DegEstimator 想提取"退化特征"（块效应、振铃、模糊）。

虽然两者可以共存，但在极限情况下可能存在竞争——卷积核的有限 capacity 需要在"内容"和"退化"之间分配。

## 十、总结

DegEstimator 是 B 方案中最小却最关键的模块。它用 **~0.1M 参数**（占总参数 0.8%）、**一次前向**、**无预训练**的代价，为整个网络提供了退化感知能力。

| 特性 | 实现 |
|------|------|
| 输入 | 原始 RGB 帧 (B, 3, H, W) |
| 输出 | 退化嵌入向量 (B, 64) |
| 结构 | 3层下采样 CNN + GAP + FC |
| 下采样率 | 1/8（空间）|
| 参数量 | ~102K (~0.10M) |
| 训练方式 | 端到端联合训练，不预训练，不冻结 |
| 条件化下游 | 8 个 FiLM 层（Bottleneck 2 + Decoder 6）|

**它的设计精髓在于"轻量"和"联合"**：
- **轻量**：不喧宾夺主，不占计算，不拖速度。
- **联合**：不是外部插件，而是网络自身进化出来的一部分——它知道什么样的退化信息对修复有用，因为它直接接收修复任务的梯度反馈。

从测试结果来看，DegEstimator 确实让 B 方案在低 QP 场景下展现了一定的相对优势（~10%），但在高 QP 场景下未能突破瓶颈。这提示我们：**在 MFQEv2 任务中，"知道退化程度"只是第一步，"如何根据退化程度精准修复"才是更难的问题**。当前通道级 FiLM 的调节粒度，可能是制约性能上限的瓶颈。
