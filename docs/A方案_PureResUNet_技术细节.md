## 1. 整体定位

PureResUNet 是本项目 A 方案的核心网络，定位是**纯卷积驱动的残差 U-Net 基线（Baseline）**。其设计哲学极为简洁：

- **零注意力机制**：不使用自注意力、通道注意力、空间注意力等任何显式注意力模块
- **零频域/小波增强**：不使用 DCT、DWT、傅里叶变换等频域操作
- **零条件化/Prompt**：不引入 QP 先验、退化图、外部提示等任何条件输入
- **零 Transformer**：不引入多头自注意力（MHSA）或前馈网络（FFN）

网络完全依靠 **卷积的局部感受野 + 残差连接 + U-Net 的多尺度编码器-解码器结构** 来学习从压缩帧到原始帧的映射。

**核心假设**：如果网络足够深、残差连接足够多，卷积本身就能隐式编码不同压缩强度（QP）的退化模式，无需显式告知网络当前是 QP22 还是 QP42。

## 2. 宏观架构概览

```
输入图像 (B, 3, H, W)
    │
    ▼
┌─────────────────────────────────────┐
│  初始卷积: Conv(3→32, 3×3)          │  ← 将 RGB 映射到特征空间
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│           编码器 (Encoder)           │
│  4 层下采样，每步 ×1/2 分辨率        │
│  通道: 32 → 64 → 128 → 256          │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│           瓶颈层 (Bottleneck)        │
│  2 × ResBlock(256)                  │
│  分辨率保持 1/16，通道 256           │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│           解码器 (Decoder)           │
│  4 层上采样，每步 ×2 分辨率          │
│  通道: 256 → 128 → 64 → 32          │
│  Skip Connection 逐层拼接融合        │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  输出头: Conv(32→3, 3×3)            │  ← 将特征映射回 RGB
└─────────────────────────────────────┘
    │
    ▼
全局残差: Output = Network_Output + Input   ← 网络只学"差异"
```

## 3. 输入图像的完整旅程

以一张 **256×256 的 RGB 图像**（训练时的 patch 尺寸）为例，逐层追踪它在网络中的尺寸、通道数和操作。

### Stage 0: 初始特征投影

```
Input:  (B, 3, 256, 256)
        │
        ▼ Conv2d(3→32, kernel=3, stride=1, padding=1, bias=False)
        │
Output: (B, 32, 256, 256)
```

> 作用：将 3 通道的像素空间映射到 32 维的特征空间，为后续残差块提供初始表示。

---

### Stage 1: 编码器第 1 层 (Encoder Level 1)

```
Input:  (B, 32, 256, 256)
        │
        ├── ResBlock(32) ──→ (B, 32, 256, 256)
        │
        └── ResBlock(32) ──→ (B, 32, 256, 256)   ← Skip-1 保存此特征
        │
        ▼ Downsample: Conv2d(32→64, kernel=3, stride=2, padding=1, bias=False)
        │
Output: (B, 64, 128, 128)
```

**下采样方式**：stride=2 的 3×3 卷积，同时完成**降分辨率**和**升通道数**。

> 注意：这里不使用 MaxPool，而是用带步长的卷积，因为卷积有参数可学习，能自适应选择下采样时的重要信息。

---

### Stage 2: 编码器第 2 层 (Encoder Level 2)

```
Input:  (B, 64, 128, 128)
        │
        ├── ResBlock(64) ──→ (B, 64, 128, 128)
        │
        └── ResBlock(64) ──→ (B, 64, 128, 128)   ← Skip-2 保存此特征
        │
        ▼ Downsample: Conv2d(64→128, kernel=3, stride=2, padding=1)
        │
Output: (B, 128, 64, 64)
```

---

### Stage 3: 编码器第 3 层 (Encoder Level 3)

```
Input:  (B, 128, 64, 64)
        │
        ├── ResBlock(128) ──→ (B, 128, 64, 64)
        │
        └── ResBlock(128) ──→ (B, 128, 64, 64)   ← Skip-3 保存此特征
        │
        ▼ Downsample: Conv2d(128→256, kernel=3, stride=2, padding=1)
        │
Output: (B, 256, 32, 32)
```

---

### Stage 4: 编码器第 4 层 (Encoder Level 4)

```
Input:  (B, 256, 32, 32)
        │
        ├── ResBlock(256) ──→ (B, 256, 32, 32)
        │
        └── ResBlock(256) ──→ (B, 256, 32, 32)   ← Skip-4 保存此特征
        │
        ▼ Downsample: Conv2d(256→256, kernel=3, stride=2, padding=1)
        │
Output: (B, 256, 16, 16)
```

> 第 4 层下采样后通道保持 256（不继续翻倍），防止参数量爆炸。

---

### Stage 5: 瓶颈层 (Bottleneck)

```
Input:  (B, 256, 16, 16)
        │
        ├── ResBlock(256) ──→ (B, 256, 16, 16)
        │
        └── ResBlock(256) ──→ (B, 256, 16, 16)
        │
Output: (B, 256, 16, 16)
```

> 瓶颈层是网络感受野最大的区域（1/16 分辨率），负责编码全局语义信息和高级退化特征。这里不再下采样，保持 16×16 的空间维度，让卷积有足够大的感受野理解整帧图像的上下文。

---

### Stage 6: 解码器第 1 层 (Decoder Level 1，对应最深层)

```
Input:  (B, 256, 16, 16)
        │
        ▼ Upsample: Bilinear(scale_factor=2) + Conv2d(256→256, kernel=3, padding=1)
        │
        (B, 256, 32, 32)
        │
        ◄── Skip-4: (B, 256, 32, 32)   [来自 Encoder Level 4]
        │
        ▼ Concat → (B, 512, 32, 32)
        │
        ▼ Conv2d(512→256, kernel=3, padding=1)
        │
        ├── ResBlock(256) ──→ (B, 256, 32, 32)
        │
        └── ResBlock(256) ──→ (B, 256, 32, 32)
        │
Output: (B, 256, 32, 32)
```

**上采样方式**：先 `bilinear` 插值放大 2 倍，再用 3×3 卷积调整通道数。这种"插值+卷积"的组合比转置卷积（Transposed Conv）更稳定，不易产生棋盘格伪影。

**Skip Connection 融合**：解码器上采样后的特征与编码器同层特征在通道维度拼接（Concat），再用 3×3 卷积将拼接后的通道压缩回目标通道数。

---

### Stage 7: 解码器第 2 层 (Decoder Level 2)

```
Input:  (B, 256, 32, 32)
        │
        ▼ Upsample: Bilinear×2 + Conv2d(256→128, kernel=3, padding=1)
        │
        (B, 128, 64, 64)
        │
        ◄── Skip-3: (B, 128, 64, 64)   [来自 Encoder Level 3]
        │
        ▼ Concat → (B, 256, 64, 64)
        │
        ▼ Conv2d(256→128, kernel=3, padding=1)
        │
        ├── ResBlock(128) ──→ (B, 128, 64, 64)
        │
        └── ResBlock(128) ──→ (B, 128, 64, 64)
        │
Output: (B, 128, 64, 64)
```

---

### Stage 8: 解码器第 3 层 (Decoder Level 3)

```
Input:  (B, 128, 64, 64)
        │
        ▼ Upsample: Bilinear×2 + Conv2d(128→64, kernel=3, padding=1)
        │
        (B, 64, 128, 128)
        │
        ◄── Skip-2: (B, 64, 128, 128)   [来自 Encoder Level 2]
        │
        ▼ Concat → (B, 128, 128, 128)
        │
        ▼ Conv2d(128→64, kernel=3, padding=1)
        │
        ├── ResBlock(64) ──→ (B, 64, 128, 128)
        │
        └── ResBlock(64) ──→ (B, 64, 128, 128)
        │
Output: (B, 64, 128, 128)
```

---

### Stage 9: 解码器第 4 层 (Decoder Level 4，最浅层)

```
Input:  (B, 64, 128, 128)
        │
        ▼ Upsample: Bilinear×2 + Conv2d(64→32, kernel=3, padding=1)
        │
        (B, 32, 256, 256)
        │
        ◄── Skip-1: (B, 32, 256, 256)   [来自 Encoder Level 1]
        │
        ▼ Concat → (B, 64, 256, 256)
        │
        ▼ Conv2d(64→32, kernel=3, padding=1)
        │
        ├── ResBlock(32) ──→ (B, 32, 256, 256)
        │
        └── ResBlock(32) ──→ (B, 32, 256, 256)
        │
Output: (B, 32, 256, 256)
```

---

### Stage 10: 输出头 + 全局残差

```
Input:  (B, 32, 256, 256)
        │
        ▼ Conv2d(32→3, kernel=3, padding=1, bias=False)
        │
Output: (B, 3, 256, 256)   ← 这是网络预测的"残差/差异"
        │
        ▼
   + Input(原始输入)         ← 全局残差连接
        │
Final: (B, 3, 256, 256)    ← 最终重建帧
```

> **全局残差学习（Global Residual Learning）** 是本网络的关键设计。网络不直接预测清晰的图像，而是预测 **"压缩帧与原始帧之间的差异"**。这种设计大大减轻了网络的学习负担——当压缩质量已经不错（如 QP22）时，差异很小，网络只需做微调；当压缩质量差（如 QP42）时，差异大，网络学习如何修复。如果没有全局残差，网络需要从零开始重建整张图像，难度高得多。

## 4. ResBlock：网络的"原子"

A 方案的每一个特征变换都由 **Pre-Activation ResBlock** 完成。它是网络中最小、最重复的构建单元。

### 4.1 结构图

```
输入特征 x (B, C, H, W)
    │
    ├──► [主分支 Body]
    │        │
    │        ▼ BatchNorm2d(C)
    │        ▼ ReLU(inplace=True)
    │        ▼ Conv2d(C→C, kernel=3, padding=1, bias=False)
    │        │
    │        ▼ BatchNorm2d(C)
    │        ▼ ReLU(inplace=True)
    │        ▼ Conv2d(C→C, kernel=3, padding=1, bias=False)
    │        │
    │        ▼ = body_out
    │
    └──► [恒等分支]  x
         │
         ▼ 逐元素相加: x + body_out
         │
         ▼ = 输出
```

### 4.2 关键设计选择

| 设计 | 具体实现 | 理由 |
|------|---------|------|
| **Pre-Activation** | BN → ReLU → Conv（而非 Conv → BN → ReLU） | 让残差分支保持"纯净"的恒等映射，梯度回传更顺畅，训练更深网络时不易梯度消失 |
| **无瓶颈设计** | 两个 3×3 卷积，通道数不变 | 保持特征表达能力，不为了降参数量而牺牲精度 |
| **无 bias** | 所有 Conv 的 `bias=False` | BatchNorm 本身带有可学习的偏移量（β），Conv 的 bias 是冗余的，去掉可减少参数量 |
| **3×3 卷积核** | kernel=3, padding=1 | 标准选择，感受野适中，计算效率高 |
| **通道不变** | 输入 C → 输出 C | 残差连接要求输入输出同维度，可直接相加 |

### 4.3 数学表达

$$
\text{ResBlock}(x) = x + \mathcal{F}(x; \{W_i\})
$$

其中 $\mathcal{F}$ 是主分支（两个 Conv-BN-ReLU 的复合函数），$\{W_i\}$ 是可学习参数。

## 5. Skip Connection 的融合机制

U-Net 的灵魂在于 Skip Connection——把编码器浅层的高分辨率细节直接传给解码器，弥补深层特征在多次下采样中丢失的空间细节。

### 5.1 融合流程

```
解码器上采样后的特征:  (B, D, H, W)
编码器同层保存的特征:  (B, S, H, W)
            │
            ▼ Concat(dim=1)
            (B, D+S, H, W)
            │
            ▼ Conv2d(D+S → D, kernel=3, padding=1, bias=False)
            (B, D, H, W)
            │
            ▼ ResBlock(D)
            (B, D, H, W)
            │
            ▼ ResBlock(D)
            (B, D, H, W)
```

### 5.2 各层具体拼接维度

| 解码器层 | 上采样后通道 D | Skip 通道 S | 拼接后 D+S | 融合 Conv 后 |
|---------|--------------|------------|-----------|------------|
| Level 1 (最深) | 256 | 256 | 512 | 256 |
| Level 2 | 128 | 128 | 256 | 128 |
| Level 3 | 64 | 64 | 128 | 64 |
| Level 4 (最浅) | 32 | 32 | 64 | 32 |

> 这种"对称式"的通道设计（D = S）让拼接后的通道数始终是 2D，融合 Conv 再压缩回 D，保证了计算量的平衡。

## 6. 完整网络层级统计

以 `base_ch=32`（默认配置）为例，统计所有可学习层。

### 6.1 模块清单

| 模块 | 数量 | 每层内部结构 |
|------|------|------------|
| 初始卷积 | 1 | Conv(3→32, 3×3) |
| 编码器 ResBlock | 8 | 4 层 × 2 个 ResBlock |
| 下采样卷积 | 4 | Conv(stride=2) |
| 瓶颈 ResBlock | 2 | 2 × ResBlock(256) |
| 上采样块 | 4 | Bilinear + Conv(3×3) |
| 解码器融合卷积 | 4 | Conv(拼接后→目标通道, 3×3) |
| 解码器 ResBlock | 8 | 4 层 × 2 个 ResBlock |
| 输出卷积 | 1 | Conv(32→3, 3×3) |

**总计**：
- **卷积层总数**：约 45 个（每个 ResBlock 含 2 个 Conv，共 18 个 ResBlock → 36 个 Conv，加上 init/down/up/fusion/out 共 9 个 Conv）
- **BatchNorm 层总数**：36 个（每个 ResBlock 含 2 个 BN）
- **ReLU 激活**：36 个（inplace）
- **Concat 操作**：4 次（Skip Connection）
- **逐元素加法**：19 次（18 个 ResBlock 残差 + 1 次全局残差）

### 6.2 参数量明细

| 组件 | 参数量 |
|------|--------|
| 初始卷积 + 编码器 | ~3.5M |
| 下采样卷积 | ~0.4M |
| 瓶颈层 | ~1.2M |
| 上采样 + 解码器融合 | ~3.5M |
| 解码器 ResBlock | ~3.0M |
| 输出卷积 | ~0.008M |
| **总计** | **~12.16M** |

> 全部参数均可学习，无常数表、无位置编码、无预训练嵌入。

## 7. 输入输出规格

### 7.1 训练模式

```
输入:  压缩帧的 256×256 随机裁剪 patch
       shape: (batch_size, 3, 256, 256)
       值域: [0.0, 1.0] (float32, 归一化到 0~1)
       
输出:  增强后的 patch
       shape: (batch_size, 3, 256, 256)
       值域: [0.0, 1.0] (clamp 后)
       
监督:  对应原始帧的相同位置 patch
       loss = L1(pred, gt) + SSIM(pred, gt)
```

### 7.2 推理/验证模式

```
输入:  整帧压缩 YUV（任意分辨率，如 1920×1080）
       shape: (1, 3, H, W)
       
处理:  如果 H 或 W > 720/1280，使用 tile-based 分块推理
       否则直接整帧前向传播（先 pad 到 16 的倍数）
       
输出:  整帧增强 YUV
       shape: (1, 3, H, W)
```

## 8. 与图中 PromptCIR 的核心差异回顾

| 维度 | A 方案 PureResUNet | PromptCIR（图中） |
|------|-------------------|------------------|
| **基本单元** | Pre-Activation ResBlock（纯卷积） | RHAG + Transformer Block（含注意力） |
| **注意力** | ❌ 无 | ✅ 自注意力 + 混合注意力 |
| **Transformer** | ❌ 无 | ✅ 深层使用 Transformer |
| **条件化/Prompt** | ❌ 无 | ✅ DPM + PIM + Prompt Block |
| **频域操作** | ❌ 无 | ❌ 无（图中也无） |
| **Skip 融合** | Concat + Conv | Concat + Conv（相同） |
| **全局残差** | ✅ 有 | ✅ 有（相同） |
| **下采样** | Stride-2 Conv | 图中未明确，通常也是 Stride Conv |
| **上采样** | Bilinear + Conv | 图中 ↑×2，通常也是插值+Conv |
| **参数量** | ~12.2M | 通常 >20M（因注意力/Transformer） |

## 9. 设计哲学总结

PureResUNet 的设计可以用一句话概括：

> **"用尽可能少的组件，做尽可能完整的事。"**

它放弃了注意力、Transformer、Prompt 等所有"花哨"的模块，只保留最经典的三个要素：
1. **卷积** —— 局部特征提取
2. **残差** —— 梯度畅通、训练稳定
3. **U-Net 多尺度** —— 细节与语义兼顾

这种极简主义的好处是：
- **可解释性强**：每一层在做什么很清晰
- **训练稳定**：没有注意力带来的数值不稳定问题
- **推断速度快**：纯卷积在 GPU 上高度优化
- **易于扩展**：骨架干净，加模块（如 B 方案的小波增强）很方便

代价是：
- **表达能力上限较低**：面对极端退化或复杂混合退化时，纯卷积可能力不从心
- **无显式退化感知**：网络必须隐式学习 QP 和退化模式的关系，对盲 QP 泛化能力天然弱于 PromptCIR 这类显式条件化网络

这也是为什么本项目在 A 方案之上，还设计了 B 方案（WaveletResUNet）—— 在保持纯卷积骨架的前提下，通过频域增强来提升细节恢复能力，而不引入注意力的复杂度和参数量膨胀。
