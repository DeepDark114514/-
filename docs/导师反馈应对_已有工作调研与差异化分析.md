## 一、调研结论：视频CAR领域确实已有频域/小波方案

经过系统调研，视频压缩伪影去除（Video Compression Artifact Removal, CAR）领域中，确实已有以下高频/频域相关工作：

### 1. MW-GAN (ECCV 2020 + TCSVT 2021) —— 最相关已有工作

- **单位**：北京航空航天大学 + 牛津大学
- **论文**：*Multi-level Wavelet-based Generative Adversarial Network for Perceptual Quality Enhancement of Compressed Video*
- **核心方法**：
  - 基于 **multi-level wavelet packet transform (WPT)** 的 GAN
  - 生成器包含 motion compensation + wavelet reconstruction (WDRB)
  - 判别器也是 wavelet-based， adversarial loss 通过 WPT 施加
  - **多帧输入**（2N+1 帧），需要相邻帧做运动补偿
- **目标**：感知质量增强（perceptual quality），非客观质量
- **参数规模**：大型 GAN，参数量远超过 13M

### 2. Xu et al. (CVPRW / NTIRE 2021)

- **单位**：复旦大学 + Bilibili
- **论文**：*Boosting the Performance of Video Compression Artifact Reduction with Reference Frame Proposals and Frequency Domain Information*
- **核心方法**：
  - **FFT loss**（损失函数层面，非架构层面）
  - 多帧方法，reference frame proposal 策略
  - NTIRE 2021 Track 1 和 Track 2 冠军

### 3. ARMoE (AAAI 2025)

- **任务**：**图像**压缩伪影去除（非视频）
- **核心方法**：DWT 作为 MoE（混合专家）系统中的一个 expert
- **与我们的关系**：任务不同（图像 vs 视频），方法不同（MoE vs UNet skip增强）

### 4. 其他相关

- 图像恢复领域（去噪/超分/去雨）有大量小波方法（见 GitHub Awesome-Image-Prior 列表）
- 但这些任务与视频CAR有本质区别

## 二、我们的方案 vs 已有工作的精确对比

| 维度 | MW-GAN (ECCV 2020) | Xu et al. (CVPRW 2021) | **我们的 MW-ResUNet** |
|------|-------------------|----------------------|----------------------|
| **任务** | 视频CAR | 视频CAR | 视频CAR |
| **输入** | 多帧 (2N+1) | 多帧 | **单帧** |
| **架构** | GAN + WPT 全图重建 | CNN + 多帧融合 | **纯 UNet + Skip WHFEM** |
| **频域位置** | 生成器+判别器**全图** | **损失函数** (FFT loss) | **Skip connection 内部** |
| **频域操作** | Multi-level WPT | FFT loss (仅损失) | **单层 Haar DWT + Depthwise Conv** |
| **参数规模** | 大型 GAN | 未明确 | **13M (轻量)** |
| **训练目标** | 感知质量 (GAN) | 客观+感知 | **客观质量 (PSNR/SSIM)** |
| **设计依据** | 启发式观察 | 启发式 | **v2 缺陷分析驱动** |

### 关键差异化总结

1. **单帧 vs 多帧**：MW-GAN 和 Xu et al. 都需要相邻帧信息，我们是**单帧方法**，无需光流估计或运动补偿，部署更简单。

2. **Skip connection 频域增强 vs 全图频域重建**：
   - MW-GAN 在**全图**上做 WPT 重建，计算量大
   - 我们只在 **UNet 的 skip connection** 中插入轻量 DWT 模块，参数量增加 < 7%
   - 这是一个**架构微观设计**，而非宏观频域方法

3. **轻量纯 CNN vs GAN**：MW-GAN 是 GAN-based，训练不稳定，参数量大。我们是纯 CNN，13M 参数，可实时处理。

4. **数据驱动设计**：我们通过系统的 v2 缺陷分析（Edge_MAE + HF_Loss + LocalStd_MSE + Chromatic_MAE）定位了边缘模糊和纹理丢失，然后**针对性**设计 WHFEM。这不是盲目跟风添加小波，而是有数据支撑的定向改进。

## 三、导师担忧的核心与应对策略

导师的担忧有两个层面：

### 层面1："频域增强"这个大方向上已有大量工作

这是事实。小波/DWT 在图像恢复领域（去噪/超分/去雨）已经被广泛研究。但：
- **视频CAR是一个更具体的子领域**，在这个子领域中，小波方法确实不多（只有 MW-GAN 系列和 Xu et al. 的 FFT loss）
- 我们的方法在**具体实现位置**（skip connection）和**方法形态**（单帧轻量）上与所有已有工作都不同

### 层面2：本科论文不需要追求 SOTA，但需要展示扎实的基础能力

导师的潜台词可能是：不要追求过于 fancy 的方法，而是要把"缺陷分析 -> 定位问题 -> 针对性改进 -> 实验验证"这个闭环做扎实。

**应对策略**：

1. **在论文中诚实讨论已有工作**：专门设一节 Related Work，明确讨论 MW-GAN 和 Xu et al.，并清晰界定与我们的区别
2. **把创新点重新包装**：
   - 不是"我们提出了一个小波增强方法"
   - 而是"**基于系统缺陷分析，我们发现边缘模糊和纹理丢失是核心问题，然后在 UNet skip connection 中设计了轻量的频域增强模块来针对性解决**"
3. **强调可解释性**：DWT 分解后 LL/LH/HL/HH 的分别处理行为透明，WHFEM 的 gamma 初始化为 0 保证训练稳定性——这些工程细节体现了扎实性

## 四、建议：是否需要调整方案？

### 选项 A：坚持当前架构，强化叙事（推荐）

- 保留 MW-ResUNet（B方案）
- 论文重点放在"**缺陷分析驱动设计**"
- 在 Related Work 中诚实对比 MW-GAN 和 Xu et al.
- 增加消融实验：证明 WHFEM 对 Edge_MAE / HF_Loss / LocalStd_MSE 的改善

**优势**：改动最小，已有工作不浪费  
**风险**：如果审稿人/答辩专家认为差异化不够，可能被challenge

### 选项 B：联合优化——架构 + 损失函数

- 保留 WHFEM（架构改进）
- **额外增加损失函数改进**：比如 Xu et al. 的 FFT loss，或者 Sobel 梯度损失
- 这样创新点更丰富，即使架构层面被说"类似 MW-GAN"，损失层面也有新东西

**优势**：创新点double，抗重复能力强  
**劣势**：工作量增加，需要重新训练

### 选项 C：换一个非频域的改进方向

- 比如时序建模（利用相邻帧）、注意力机制、知识蒸馏等
- 但v2缺陷分析已经定位了高频丢失，非频域方法针对性不强

**优势**：完全避开"频域重复"风险  
**劣势**：与缺陷分析结论脱节，需要重新做大量调研

## 五、给导师的回复草稿

> 老师，关于高频方案可能重复的顾虑，我专门调研了已有工作。视频CAR领域中，确实已有 MW-GAN (ECCV 2020) 用 multi-level wavelet packet transform 做感知质量增强，以及 Xu et al. (CVPRW 2021) 用 FFT loss 做高频恢复。
>
> 但我们的方案与它们有本质区别：
> 1. **MW-GAN 是多帧+GAN方案**，需要运动补偿，参数量大；我们是**单帧纯CNN**，13M参数，即插即用
> 2. **MW-GAN 在全图层面做 WPT 重建**，我们只在 **UNet skip connection 内部**做轻量 Haar DWT 增强，参数量增加 < 7%
> 3. Xu et al. 是在**损失函数层面**用 FFT，我们是在**架构层面**改进
> 4. 最关键的是，我们的设计不是跟风，而是基于之前系统的 v2 缺陷分析，**数据驱动地定位了边缘模糊和纹理丢失**，然后针对性设计 WHFEM
>
> 如果老师觉得仍有重复风险，我可以考虑在保留 WHFEM 架构的同时，增加损失函数层面的改进（如 FFT loss 或 Sobel 梯度损失），形成"架构+损失"联合优化，这样创新点会更丰富。
>
> 请老师定夺。

## 六、结论

导师的担忧是有道理的——频域方法确实不新。但我们的方案在**具体实现位置**（skip connection）、**方法形态**（单帧轻量）、**设计依据**（缺陷分析驱动）上与所有已有工作都不同。

**建议**：坚持当前方向，但强化叙事——重点不是"我们用了小波"，而是"**基于缺陷分析，我们在 skip connection 中做了轻量频域增强来针对性解决边缘模糊和纹理丢失**"。

如果导师仍有顾虑，可增加损失函数改进作为保险。
