## 1. 整体宏观结构

```
Input (3, 256, 256)
    |
    v
[Init Conv] 3 -> 32
    |
    +---> [Enc1] 2xResBlock(32) ---> Skip1 (32ch, 256x256) ---> Down ---> (64ch, 128x128)
    |                                                              |
    +---> [Enc2] 2xResBlock(64) ---> Skip2 (64ch, 128x128) ---> Down ---> (128ch, 64x64)
    |                                                              |
    +---> [Enc3] 2xResBlock(128) --> Skip3 (128ch, 64x64) ---> Down ---> (256ch, 32x32)
    |                                                              |
    +---> [Enc4] 2xResBlock(256) --> Skip4 (256ch, 32x32) ---> Down ---> (256ch, 16x16)
                                                                            |
                                                                    [Bottleneck]
                                                                    2xResBlock(256)
                                                                            |
    +-----------------------------------------------------------------------+
    |
    v
[Dec1] Up(256ch) + Skip4(256ch) -> Concat(512ch) -> Conv(256ch) -> 2xResBlock(256) -> (256ch, 32x32)
    |
[Dec2] Up(128ch) + Skip3(128ch) -> Concat(256ch) -> Conv(128ch) -> 2xResBlock(128) -> (128ch, 64x64)
    |
[Dec3] Up(64ch)  + Skip2(64ch)  -> Concat(128ch) -> Conv(64ch)  -> 2xResBlock(64)  -> (64ch, 128x128)
    |
[Dec4] Up(32ch)  + Skip1(32ch)  -> Concat(64ch)  -> Conv(32ch)  -> 2xResBlock(32)  -> (32ch, 256x256)
    |
    v
[Output] Conv(32 -> 3) -> (3, 256, 256)
```

## 2. 编码器（Encoder）

编码器使命：**逐层提取多尺度特征，同时降低空间分辨率、提升通道数**。

每一层编码器由三部分组成：
1. `2xResBlock`：在**当前分辨率**下进行特征精炼（不改变尺寸）
2. `Skip 保存`：将精炼后的特征存入 skip list，供解码器使用
3. `Down`：stride=2 的 3x3 Conv，空间尺寸减半、通道数提升

### 2.1 初始卷积

| 步骤 | 操作 | 输入尺寸 | 输出尺寸 | 通道变化 |
|------|------|---------|---------|---------|
| Init Conv | `Conv2d(3, 32, 3, padding=1)` | 256x256 | 256x256 | 3 -> **32** |

- 不改变空间尺寸（padding=1, stride=1）
- 将 3 通道 RGB 映射到基础通道数 32

### 2.2 Enc1（第一层编码器）

| 步骤 | 操作 | 空间尺寸 | 通道数 | 说明 |
|------|------|---------|--------|------|
| ResBlock x2 | `BN-ReLU-Conv-BN-ReLU-Conv` | 256x256 | 32 | 特征精炼，尺寸不变 |
| **Skip1 保存** | `skips.append(x)` | 256x256 | **32** | 存入 skip list，供 Dec4 使用 |
| Down | `Conv2d(32, 64, 3, stride=2)` | 256x256 -> **128x128** | 32 -> **64** | 空间减半，通道翻倍 |

### 2.3 Enc2（第二层编码器）

| 步骤 | 操作 | 空间尺寸 | 通道数 | 说明 |
|------|------|---------|--------|------|
| ResBlock x2 | `BN-ReLU-Conv-BN-ReLU-Conv` | 128x128 | 64 | 特征精炼 |
| **Skip2 保存** | `skips.append(x)` | 128x128 | **64** | 供 Dec3 使用 |
| Down | `Conv2d(64, 128, 3, stride=2)` | 128x128 -> **64x64** | 64 -> **128** | 空间减半，通道翻倍 |

### 2.4 Enc3（第三层编码器）

| 步骤 | 操作 | 空间尺寸 | 通道数 | 说明 |
|------|------|---------|--------|------|
| ResBlock x2 | `BN-ReLU-Conv-BN-ReLU-Conv` | 64x64 | 128 | 特征精炼 |
| **Skip3 保存** | `skips.append(x)` | 64x64 | **128** | 供 Dec2 使用 |
| Down | `Conv2d(128, 256, 3, stride=2)` | 64x64 -> **32x32** | 128 -> **256** | 空间减半，通道翻倍 |

### 2.5 Enc4（第四层编码器）

| 步骤 | 操作 | 空间尺寸 | 通道数 | 说明 |
|------|------|---------|--------|------|
| ResBlock x2 | `BN-ReLU-Conv-BN-ReLU-Conv` | 32x32 | 256 | 特征精炼 |
| **Skip4 保存** | `skips.append(x)` | 32x32 | **256** | 供 Dec1 使用 |
| Down | `Conv2d(256, 256, 3, stride=2)` | 32x32 -> **16x16** | 256 -> **256** | 空间减半，通道不变 |

> **注意**：Enc4 的 Down 比较特殊，输入输出通道都是 256（`8*base_ch`）。如果继续翻倍会到 512，参数量和计算量都会失控，所以设计者在最深层保持了通道数不变。

## 3. 瓶颈层（Bottleneck）

瓶颈层位于 U 型结构的最底部，负责在**最低分辨率、最高语义层级**上进行特征变换。

| 步骤 | 操作 | 空间尺寸 | 通道数 | 说明 |
|------|------|---------|--------|------|
| ResBlock x2 | `BN-ReLU-Conv-BN-ReLU-Conv` | 16x16 | 256 | 在最小特征图上做最深层的特征变换 |

- 不保存 skip（因为它是最底层，没有对应的编码器层）
- 输出直接送入 Dec1 的上采样模块

---

## 4. 解码器（Decoder）

解码器使命：**逐层恢复空间分辨率，同时融合编码器的多尺度 skip 特征**。

每一层解码器由四部分组成：
1. `Up`：Bilinear 插值放大 2 倍 + Conv 降通道
2. `Concat`：与对应层的 skip 特征在通道维度拼接
3. `Conv 融合`：3x3 Conv 将拼接后的双倍通道降回目标通道数
4. `2xResBlock`：在融合后的特征上进行精炼

解码器取 skip 的顺序是**从深到浅**（`skips[3], skips[2], skips[1], skips[0]`），即：
- Dec1 取 Enc4 的 skip（最深、语义最强）
- Dec4 取 Enc1 的 skip（最浅、纹理最丰富）

### 4.1 Dec1（第一层解码器）

| 步骤 | 操作 | 空间尺寸 | 通道数 | 说明 |
|------|------|---------|--------|------|
| Up | `Upsample(x2)` + `Conv(256->256)` | 16x16 -> **32x32** | 256 | 先放大，再保持通道 |
| Concat | `cat([up, Skip4], dim=1)` | 32x32 | 256 + 256 = **512** | 拼接 Enc4 的 skip |
| Conv 融合 | `Conv2d(512, 256, 3)` | 32x32 | 512 -> **256** | 把双倍通道压回去 |
| ResBlock x2 | `BN-ReLU-Conv-BN-ReLU-Conv` | 32x32 | 256 | 精修融合特征 |

### 4.2 Dec2（第二层解码器）

| 步骤 | 操作 | 空间尺寸 | 通道数 | 说明 |
|------|------|---------|--------|------|
| Up | `Upsample(x2)` + `Conv(256->128)` | 32x32 -> **64x64** | 256 -> **128** | 放大并降通道 |
| Concat | `cat([up, Skip3], dim=1)` | 64x64 | 128 + 128 = **256** | 拼接 Enc3 的 skip |
| Conv 融合 | `Conv2d(256, 128, 3)` | 64x64 | 256 -> **128** | 降通道 |
| ResBlock x2 | `BN-ReLU-Conv-BN-ReLU-Conv` | 64x64 | 128 | 精修 |

### 4.3 Dec3（第三层解码器）

| 步骤 | 操作 | 空间尺寸 | 通道数 | 说明 |
|------|------|---------|--------|------|
| Up | `Upsample(x2)` + `Conv(128->64)` | 64x64 -> **128x128** | 128 -> **64** | 放大并降通道 |
| Concat | `cat([up, Skip2], dim=1)` | 128x128 | 64 + 64 = **128** | 拼接 Enc2 的 skip |
| Conv 融合 | `Conv2d(128, 64, 3)` | 128x128 | 128 -> **64** | 降通道 |
| ResBlock x2 | `BN-ReLU-Conv-BN-ReLU-Conv` | 128x128 | 64 | 精修 |

### 4.4 Dec4（第四层解码器）

| 步骤 | 操作 | 空间尺寸 | 通道数 | 说明 |
|------|------|---------|--------|------|
| Up | `Upsample(x2)` + `Conv(64->32)` | 128x128 -> **256x256** | 64 -> **32** | 放大回原图尺寸 |
| Concat | `cat([up, Skip1], dim=1)` | 256x256 | 32 + 32 = **64** | 拼接 Enc1 的 skip |
| Conv 融合 | `Conv2d(64, 32, 3)` | 256x256 | 64 -> **32** | 降通道 |
| ResBlock x2 | `BN-ReLU-Conv-BN-ReLU-Conv` | 256x256 | 32 | 精修 |

> **关键观察**：Dec4 的输出通道数回到了 `base_ch=32`，空间尺寸回到了 `256x256`，与 Init Conv 的输出完全一致。这为最终的输出头提供了最自然的特征表示。

## 5. Skip Connection 流向图

```
                Enc1 output (32ch, 256x256)
                     |
                     |  Skip1
                     |-------------------------------\
                     |                                \
                     v                                 \
[Down] -> Enc2 ...                                    \
                                                        \
                Enc2 output (64ch, 128x128)              \
                     |                                    \
                     |  Skip2                               \
                     |------------------------\             \
                     |                         \             \
                     v                          \             \
[Down] -> Enc3 ...                               \             \
                                                    \             \
                Enc3 output (128ch, 64x64)           \             \
                     |                                \             \
                     |  Skip3                          \             \
                     |-------------------\              \             \
                     |                    \              \             \
                     v                     \              \             \
[Down] -> Enc4 ...                          \              \             \
                                             \              \             \
                Enc4 output (256ch, 32x32)    \              \             \
                     |                         \              \             \
                     |  Skip4                   \              \             \
                     |---------\                 \              \             \
                     |          \                 \              \             \
                     v           \                 \              \             \
[Down] -> Bottleneck(256ch, 16x16) \                 \              \             \
                                     \                 \              \             \
[Up] -> Dec1 <-----------------------/                 \              \             \
(256ch, 32x32)                                          \              \             \
                                                         \              \             \
[Up] -> Dec2 <------------------------------------------/              \             \
(128ch, 64x64)                                                          \             \
                                                                         \             \
[Up] -> Dec3 <----------------------------------------------------------/             \
(64ch, 128x128)                                                                        \
                                                                                        \
[Up] -> Dec4 <-------------------------------------------------------------------------/
(32ch, 256x256)
```

### Skip 连接的设计意图

| Skip | 来源 | 分辨率 | 通道 | 所含信息 | 在解码器中的角色 |
|------|------|--------|------|---------|----------------|
| Skip1 | Enc1 后 | 256x256 | 32 | 纹理、边缘、细粒度伪影 | Dec4 恢复细节 |
| Skip2 | Enc2 后 | 128x128 | 64 | 局部结构、中等尺度特征 | Dec3 恢复结构 |
| Skip3 | Enc3 后 | 64x64 | 128 | 区域语义、轮廓 | Dec2 恢复区域 |
| Skip4 | Enc4 后 | 32x32 | 256 | 全局语义、整体亮度 | Dec1 恢复全局 |

## 6. 输出头

| 步骤 | 操作 | 输入尺寸 | 输出尺寸 | 通道变化 |
|------|------|---------|---------|---------|
| Output | `Conv2d(32, 3, 3, padding=1)` | 256x256 | 256x256 | 32 -> **3** |

- **无激活函数**：输出是原始像素残差 `pred ∈ (-∞, +∞)`
- 推理时在外部进行 `clamp(0, 1)`
- 如果网络学不到有用特征，输出趋近于 0，此时整体近似恒等映射 `y ≈ x`

## 8. 关键设计问答

### Q1: 为什么编码器最后一层（Enc4）Down 后通道不变（256->256）？

如果继续翻倍到 512，瓶颈层将会有 512 个通道。以 base_ch=32 计算：
- 瓶颈层 2xResBlock(512) 的参数量 = `2 * 18 * 512^2 = 9,437,184`（约 9.4M）
- 仅此一层就占总参数量的大部分，远超 5-15M 的设计目标

因此设计者在最深层将通道锁定在 `8*base_ch=256`，控制总参数量。

### Q2: 解码器的通道为什么是"对称但不完全镜像"的？

| 层级 | 编码器通道 | 解码器通道 | 是否对称 |
|------|-----------|-----------|---------|
| Enc1 / Dec4 | 32 | 32 | ✅ 对称 |
| Enc2 / Dec3 | 64 | 64 | ✅ 对称 |
| Enc3 / Dec2 | 128 | 128 | ✅ 对称 |
| Enc4 / Dec1 | 256 | 256 | ✅ 对称 |
| Bottleneck | 256 | — | — |

整体上是**对称**的，但注意：
- 编码器通道变化：`32 -> 64 -> 128 -> 256 -> 256`
- 解码器通道变化：`256 -> 256 -> 128 -> 64 -> 32`

Dec1 的输入是 bottleneck 的 256ch，输出也是 256ch，所以 Dec1 的"Up+Conv"这一步输入输出通道相同（256->256），这与 Enc4 的 Down（256->256）形成对应。

### Q3: Concat 后为什么必须跟一个 Conv？

假设去掉 Conv 融合层：
```
Up(256ch) -> cat with Skip4(256ch) -> 512ch -> ResBlock(512)
```

ResBlock(512) 的参数量 = `18 * 512^2 = 4,718,592`（约 4.7M），而且后续 Dec2 的 Up 需要从 512 降到 128，参数量进一步爆炸。

加入 `Conv(512->256)` 后：
- ResBlock 始终在当前目标通道数上运行
- 参数量可控
- 融合 Conv 可以学习如何"加权混合"上采样特征和 skip 特征

### Q4: Skip 特征取自 Down 之前还是之后？

**取自 Down 之前**。代码逻辑：
```python
for res in self.enc_blocks[i]:
    x = res(x)       # ResBlock 精炼
skips.append(x)      # <<< 保存！此时还没下采样
x = self.down_blocks[i](x)  # 下采样
```

这意味着：
- Skip4 的空间尺寸是 **32x32**（Enc4 ResBlock 后），而不是 bottleneck 的 16x16
- Dec1 上采样后也是 32x32，两者天然对齐

如果 skip 取自 Down 之后（16x16），Dec1 上采样到 32x32 后还需要额外处理 skip 的尺寸，增加了复杂性。

### Q5: 为什么用 Bilinear + Conv 而不是 ConvTranspose？

ConvTranspose2d（反卷积）在某些 stride/kernel 组合下会产生**棋盘伪影**（checkerboard artifacts），这是因为 kernel 重叠不均匀导致的。

Bilinear 插值是确定性的平滑放大，不会产生伪影。虽然计算量略高（插值 + 卷积两步），但对于视频质量增强任务，输出质量优先于计算速度。
