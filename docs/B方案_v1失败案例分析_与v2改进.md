## 一、实验背景

### 1.1 A 方案暴露的问题

A 方案（PureResUNet）Cross-QP 评估结果：

| QP | Model PSNR | Base PSNR | Gain (dB) | Model SSIM | Base SSIM | Gain |
|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
| 22 | 36.79 | 36.62 | +0.17 | 0.9384 | 0.9346 | +0.0038 |
| 27 | 34.47 | 34.22 | +0.25 | 0.9113 | 0.9055 | +0.0059 |
| 32 | 32.39 | 32.04 | +0.35 | 0.8819 | 0.8715 | +0.0104 |
| 37 | 29.74 | 29.38 | +0.36 | 0.8266 | 0.8123 | +0.0144 |
| 42 | 27.87 | 27.49 | +0.38 | 0.7825 | 0.7625 | +0.0201 |

**核心诊断**：QP32→QP42，baseline PSNR 暴跌 4.55 dB，但模型增益仅增加 0.03 dB。说明 A 方案**修复策略近似固定**，对输入质量"浑然不觉"。

### 1.2 B 方案的设计动机

给网络加一只"眼睛"（退化估计器）和一组"旋钮"（FiLM 条件层），让网络根据输入质量自适应调节修复力度。

**理论基础**：
- PromptCIR (CVPRW 2024)：隐式编码退化信息，不预测显式 QF
- FiLM (AAAI 2018)：逐通道仿射变换 γ·F + β，通用条件化层

## 二、B 方案 v1 实现细节

### 2.1 网络架构

```
DegEstimator (输入旁路)
    3层 Conv (stride=2) + GAP + FC
    通道: 16→32→48 → GAP → FC(48→64)
    参数量: ~22K

FiLMResBlock (8 个)
    Bottleneck × 2 + Decoder L0/1/2 × 2
    数学形式: Output = x + (1 + γ) · body_out + β
    
DegFiLMResUNet (总参数量)
    BaseUNet: 12.16M
    + DegEstimator: 0.02M
    + FiLM 生成器: 0.18M
    = 12.36M (+1.69%)
```

### 2.2 训练配置

| 参数 | 值 |
|------|-----|
| base_ch | 32 |
| loss | L1_SSIM (1:1) |
| optimizer | Adam, lr=1e-4 |
| scheduler | StepLR (step=30, gamma=0.1) |
| batch_size | 32 |
| train QP | [22, 32, 42] 混合 |
| val QP | 32 |
| early_stop_patience | 4 |
| val_interval | 5 epoch |

### 2.3 训练过程

- **Epoch 20**: Val PSNR = 32.7065 dB（最佳）
- **Epoch 25**: Val PSNR = 29.1186 dB（异常暴跌）
- **Epoch 40**: Val PSNR = 32.5507 dB → **早停触发**（连续 4 次验证不提升）

## 四、关键问题：增益曲线"饱和"现象

### 4.1 现象描述

从 B 方案的增益波动图可以清晰看到：

```
QP22  QP27  QP32  QP37  QP42
  │     │     │     │     │
 0.08  0.19  0.32  0.32  0.33
              ↑_________↑
              饱和区：几乎平行！
```

**理想曲线**：持续上升（QP 越高 → 修复空间越大 → 增益越大）  
**实际曲线**：QP32 之后**平着走**，增幅从 +0.13 骤降到 +0.00/+0.01

### 4.2 物理意义

这意味着 B 方案**没有实现预期的"自适应修复力度调节"**：
- QP22（轻压缩）：γ 较小，修复保守 ✅
- QP27（中压缩）：γ 中等，修复适度 ✅
- QP32（重压缩）：γ 较大，修复积极 ✅
- QP37（更重压缩）：γ 和 QP32 差不多 ❌
- QP42（最重压缩）：γ 还是差不多 ❌

模型把 QP32/37/42 当成**同一类退化**来处理了。

## 五、错因分析

### 5.1 根本原因：DegEstimator 在"重压缩区"失去区分能力

**假设的退化嵌入空间**（实际可能如此）：

| QP | 退化嵌入 d | FiLM 生成的 γ |
|:--:|:--|:--|
| 22 | [0.1, -0.2, 0.3, ...] | 较小（保守修复） |
| 27 | [0.5, 0.1, 0.7, ...] | 中等（适度修复） |
| 32 | [0.9, 0.6, 0.8, ...] | 较大（积极修复） |
| 37 | **[0.9, 0.6, 0.8, ...]** | 和 QP32 几乎一样！ |
| 42 | **[0.9, 0.6, 0.8, ...]** | 还是一样！ |

**为什么区分不出来？**

1. **压缩伪影的"视觉饱和"**
   - QP32 以上的块效应、振铃已经很严重
   - 网络难以区分"很糊"和"更糊"的微妙差异

2. **DegEstimator 容量太轻**
   - 仅 22K 参数，3 层 CNN (16→32→48)
   - GAP 直接压到 1×1，空间信息完全丢失
   - 表征空间不够精细，高 QP 样本的特征重叠

3. **训练信号的缺失**
   - 训练时只有 3 个 QP（22/32/42）
   - 模型对中间 QP（27/37）的插值能力弱
   - 没见过 QP37，只能"猜测"

### 5.2 次要原因：FiLM 调节范围有限

- 只在 8 个残差块中注入条件化
- 只调通道统计量（γ, β），没有空间维度
- 即使 d 有微小差异，生成的 γ 也可能落在同一区间

### 5.3 训练不充分

- B 方案 epoch 20 被早停，A 方案 epoch 65
- Val PSNR 在 epoch 20 后停滞，连续 4 次不提升
- 模型在 val set 上过早饱和，没充分学习退化感知

## 六、验证思路（用于区分"指示器问题"vs"调整器问题"）

### 6.1 实验设计

> 固定 DegEstimator 输出为 0（d=0），此时 FiLM 输出 γ=0, β=0，模型退化为标准 ResBlock。

**预期结果**：
- 如果性能 ≈ A 方案 → 说明 FiLM 本身没引入负面效应
- 如果性能 << A 方案 → 说明网络结构改动有问题

### 6.2 已有间接证据

B 方案 v1 的 val PSNR（32.71）≈ A 方案（32.79），差距很小（0.08 dB）。  
这说明 **FiLM 的数学形式和插入策略本身是对的**，没有破坏主干网络的学习能力。

**结论**：问题主要在 **DegEstimator（指示器）**，FiLM（调整器）是被拖累的。

## 七、改进措施（v2）

### 7.1 已实施：增大 DegEstimator 容量

| 维度 | v1 | v2 |
|------|-----|-----|
| 通道数 | 16→32→48 | **32→64→128** |
| 参数量 | ~22K | **~102K** |
| B 方案总参数 | 12.36M | **12.44M** |
| 总增量 | +1.69% | **+2.30%** |

**动机**：更大的表征空间 → 有机会区分 QP32/37/42 的微妙差异。

### 7.2 已实施：放宽早停条件

| 维度 | v1 | v2 |
|------|-----|-----|
| patience | 4 | **6** |
| 等效 epoch | 20 | **30** |

**动机**：给模型多 10 个 epoch 的收敛机会，避免过早停止。

### 7.3 待验证：如果 v2 仍失败，下一步改进

1. **给 DegEstimator 加辅助任务**
   - 加一个轻量分类头，预测 QP 等级（轻/中/重）
   - 用辅助损失强制 d 向量与 QP 单调相关

2. **DegEstimator 输出加统计特征**
   - 除了 GAP（均值），再加 GSP（全局标准差）
   - QP 直接影响像素分布的离散度，方差是有用信号

3. **FiLM  gamma 非线性化**
   - 当前 γ 是线性映射，高 QP 区域敏感度不够
   - 改为 γ = tanh(Linear(d))，让高 QP 区域的调节更敏感

4. **空间维度条件化**
   - 在 FiLM 后加轻量空间注意力（1×1 conv 生成 spatial weight）
   - PromptCIR 强调 "spatial-wise knowledge"，FiLM 缺少这一维

## 八、论文叙事价值

### 8.1 失败的正面意义

这个"饱和"现象本身就是一篇好论文的**关键发现**：

> "我们发现，单纯增加条件化模块（FiLM）并不能自动实现跨 QP 的自适应修复。问题的关键在于退化估计器能否在重压缩区（QP32+）产生单调相关的嵌入向量。如果 DegEstimator 在视觉饱和区失去区分能力，后续的条件化层再精巧也无法发挥作用。"

### 8.2 从 v1 到 v2 的叙事线

```
v1: 轻量 DegEstimator (22K) → QP32+ 饱和 → 失败
    ↓ 诊断：指示器在重压缩区"失明"
v2: 增大 DegEstimator (102K) → 表征空间更大 → 验证中
    ↓ 如果仍失败
v3: 辅助任务 + 统计特征 → 强制单调性 → 待实验
```

这条线展示了**系统性的工程迭代过程**，比直接说"我们做了 B 方案效果很好"更有说服力。

## 九、总结

| 项目 | v1 结果 | 根因 | v2 改进 |
|------|---------|------|---------|
| 整体性能 | 低于 A | 训练不充分 + 指示器弱 | 大容量 + 长 patience |
| 增益饱和 | QP32 后平着走 | DegEstimator 重压缩区"失明" | 增大表征空间 |
| 退化感知 | 有但弱 | 22K 参数不够区分高 QP | 102K 参数 |
| FiLM 效果 | 被拖累 | 输入 d 质量不高 | 先不动，等 v2 结果 |

**核心教训**：条件化网络中，"条件生成器"的质量决定了整个系统的上限。FiLM 再精巧，如果 DegEstimator 输出的 d 是"瞎子"，调节旋钮也是瞎调。
